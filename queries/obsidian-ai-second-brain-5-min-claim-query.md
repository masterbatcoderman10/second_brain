## Question

Can you really build a useful AI second brain in Obsidian in 5 minutes using Claude Code?

## Answer

**The 5-minute claim is highly contested.** The most upvoted response (125 likes) came from @aiedge_ claiming it takes only 5 minutes, but the community response and counter-examples suggest this dramatically underestimates the real setup time.

What the evidence shows:

1. **The 5-minute claim likely refers to the initial connection** — linking Claude Code to an existing Obsidian vault via tools like [[byterover-cli]] can be quick. The actual claim was specifically about the AI combo setup, not a fully functional second brain.

2. **The community contradicts the speed** — @thegreatest_sv's counter was widely understood: "we've reached the peak of the productivity loop: spending 4 hours configuring an AI Chief of Staff in Obsidian to avoid doing 30 minutes of actual work."

3. **Vault curation is the real time investment** — building a useful second brain requires curating notes, structuring them (PARA, Zettelkasten), and maintaining them over time. The 5-minute claim ignores the months of curation that precede a useful system.

4. **New users report needing YouTube tutorials** just to add a note — @OVilla_Builds said Obsidian requires "peak power user energy" and "I needed a YouTube tutorial just to add a note."

## Practical Verdict

The **initial connection** of Claude Code + Obsidian can be done in minutes using tools like [[byterover-cli]]. But building a **functional, useful second brain** that actually enhances productivity requires significant ongoing curation and learning investment. The 5-minute claim is best interpreted as marketing language rather than operational reality.

The pattern is real and powerful — see [[obsidian-ai-second-brain]] — but plan for weeks of curation, not minutes of setup.
