## How to exploit Mindshare arbitrage

If you're active across LI, X, Youtube, Instagram and Tiktok you'll notice that each platform speaks its own distinct language and at any given moment holds a set of new original ideas and thoughts that only live on that platform.

The distinct language and new ideas circulating on a platform are what occupies the mindshare of its users.

If you're very attentive to the differences between each platform (some would call this being chronically online), you can exploit the gaps between their mindshares (and get a shitton of views).

In practice I am talking about being the first to spot an emerging viral format/ idea on one platform and to translate into the language of another and posting it there. 

This is what my co-founder @JoschuaBuilds and I call mindshare arbitrage.

There's an art to stealing and without, you're just gonna end up looking like one of a thousand copycats burning whatever credibility you had. 

Most of that art comes to not being lazy...

## The System

Here's the whole thing at a glance. Six stages, two of them need my involvement, the rest runs on its own. A cron at 6 AM kicks off discovery, another at 6 PM pulls the day's analytics back into tomorrow's scoring.    

Here's how each stage works in greater detail.

## Step 1: Discovery

Every morning at 6 AM, a cron job runs. I call it the pulse.

It scrapes YouTube, X, TikTok, and changelogs for trending AI and tech content using three tools:

→ A custom Playwright + Cheerio script for static pages like the Claude Code changelog
→ Apify with the official [Ultimate Scraper skill](https://github.com/apify/agent-skills/tree/main/skills/apify-ultimate-scraper) for TikTok, Instagram, Linkedin and X  
→ for YouTube [yt-dlp](https://github.com/yt-dlp/yt-dlp) with a custom search skill and [Supadata.ai](https://supadata.ai/?ref=robin) to get the transcripts

Each idea gets scored by a simple KPI: Recency times engagement. Everything gets stored in a local SQLite and deduped by URL hash so the same topic doesn't shows up twice.

For any video it finds, it pulls the transcript automatically via [Supadata](https://supadata.ai/?ref=robin) API. By the time I sit down in the morning, the system already watched the videos and figured out what is relevant for me.

The output is a ranked list of ideas ready for the Tinder UI.

## Step 2: Morning Review

I don't want to blindly repurpose every post. Generating content is expensive, so I only want to run the pipeline on ideas that actually fit.

Therefore, I built a /review command in Claude Code first. But reviewing 20-50 posts one by one through a CLI is a pain in the ass. So I decided to build a local UI with a Tinder-like swipe. Right to keep, left to skip, star to favorite.

Every morning I open it. Yesterday's discoveries ranked by score. Source, engagement numbers, one-line summary per content idea.

It only takes me 5-10 minutes each day to go through these.

Kept ideas go into the generation queue. Starred ideas get priority. No writing, no decisions about format or platform. Just: is this idea worth turning into content?

## Step 3: Generation

Once an idea is kept, I run /generate-content. An orchestrator agent kicks off.

First, a research pass. The idea goes into NotebookLM, which produces a research brief to build on the original post. If the pulse already pulled a transcript, that gets fed in too. One research pass feeds all platforms.

Then four agents run in parallel, one per platform:

→ LinkedIn: professional but real. Carousels for tutorials, text posts for opinions.  
→ TikTok EN: casual, short slides, punchy hooks. 3-8 slides per post.  
 → TikTok DE: German adaptation of the EN content. Same images, translated text.  
 → Instagram: TikTok EN slides cropped to 1080x1350. No extra generation.

Each platform runs through the same three-step loop:  
writer → humanizer → critic.

Strips 29 documented AI writing patterns. Separately, my voice profile gets applied.  I built this flow once during onboarding from 100+ of my real X and LinkedIn posts. The writer drafts the content using the platform's voice profile. Then a [humanizer skill](https://github.com/blader/humanizer/blob/main/SKILL.md).

Then a critic agent scores the draft on hook strength, factual accuracy, and economy. Each dimension needs 8/10 to pass. If it fails, it goes back to the writer for revision. Max two rounds. If it still doesn't pass, the best version gets flagged so I check it manually.

I trigger the command, go do something else, come back to finished drafts.

## Step 4: Slides

The writers output text. But TikTok and Instagram need actual images. LinkedIn needs rendered carousel slides.

Two separate rendering tracks, determined by platform. No choosing involved.

LinkedIn branded slides: I have 1080x1350 HTML-templates in our 11x brand design. Four templates for these layout types: hook card for the first slide, numbered items for walkthroughs, bullet lists for features, and pull quotes for closers. The agent writes the slide specs as JSON and uses a Python script build out the HTML, and uses playwright-cli to screenshot each slide.

TikTok and Instagram use personal slides. These are real photos of me from my phone gallery. I manually pick them and drop them into a folder, then had Claude label each one with a description of what's in the image. The renderer matches each slide's topic to a photo using keyword overlap against those descriptions. Text gets composited on top via node-canvas. TikTok exports at 1080×1920. Instagram crops the same renders to 1080×1350.

If the content references a GitHub repo, a screenshot agent runs first and captures the repo's OG card. That image gets composited into the slides as an overlay. I use this for formats like "5 Claude Skills you MUST use".

## Step 5: Approval

Last step is approving the drafts. I built an /approve command for this with a preview UI that shows me how the post will look on each platform before it goes live.

Once I approve, the agent uses my [Postiz](https://postiz.pro/robin-sadeghpour-faraj) skill and the Postiz CLI to push it out. Uploads media, schedules the post per platform. LinkedIn publishes directly. Instagram publishes directly too. I can't attach music that way, haven't figured out a workaround. TikTok publishes to draft so I can add music manually before it goes live.

## Step 6: The Feedback Loop

Every evening a script pulls analytics from [Postiz](https://postiz.pro/robin-sadeghpour-faraj) for the day's posts. Impressions, clicks, reply count.

Those numbers become multipliers per topic and format. After a week, the system knows that Claude Code carousels get 2.8x the reach of a generic AI trend quote slide. Next morning the pulse uses those multipliers when ranking new ideas. Stuff that works floats up. Stuff that doesn't gets retired.

## How you can improve the system

The slides look the same after a while. I have 30 photos in the catalog and I rarely add new ones. The text and image overlays are hard to get right, positioning and readability depend on the photo, and you need the right images to overlay (logos, GitHub screenshots, etc).

This works for my niche because AI/tech content recycles well across platforms. If your content is more visual or more personal, the automation won't work the same way.

None of this is original. I'm not bringing my own creative thinking into most of these posts. The system finds what's trending, rewrites it in my voice, publishes it. Useful, but not thought leadership (hence the title of this article...).

For what it's worth, I planned to use this as baseline content. A pipeline that lets me publish consistently, then layer original stuff on top when I have ideas worth sharing and the time for it.

But in reality, same problem exists: I'm too lazy to do anything original and creating original content is hard and time consuming. So I'm only doing this. It's also not a magic pill that generates millions of views everyday (but it can get you posts that will pop off). 

For me personally iIt only solves one thing: consistently putting out relevant content in my niche.

## Wrapping Up

My pipeline finds ideas, writes drafts in my voice, renders slides, and schedules everything through [Postiz](https://postiz.pro/robin-sadeghpour-faraj).

If you want to build something like [this or contribute](https://github.com/robinsadeghpour/content-workflow/tree/main), the system is open source. I think every content pipeline needs at least these pieces: some kind of monitoring or pull system for ideas, a humanizer to strip the AI smell, a writing skill with your voice, a quality gate, and something like Postiz for scheduling.

The whole system is Claude Code skills + subagents, [Apify](https://apify.com), a SQLite database, and [Postiz](https://postiz.pro/robin-sadeghpour-faraj). Runs on my MacBook.
