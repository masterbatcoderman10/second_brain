How to Grow Your LinkedIn with OpenClaw: The 5-Phase Playbook Behind a 30K-Follower Account

Max Mitcham (founder of Trigify) grew his LinkedIn to 30,000+ followers running his entire content engine on AI agents. His whole SaaS works the same way. Marketing, sales, customer success, even product ops. The server is a Mac Mini sitting on his desk, and he runs everything off a stack he calls OpenClaw.

I jumped on a 60-minute call with him to break the system down step by step. So you can build the same one yourself.

The full setup took him most of this year to dial in. But the architecture is replicable. What follows is the exact phased blueprint, in the order he built it. Five phases. Each phase yields something useful before you move to the next.

If you're starting from zero, this is the path.

---

# Phase 1: The Foundation (hardware + repo)

Before you do anything with agents, you need a place to run them and a place to store everything they produce. Two pieces.

- Step 1: Get a Mac Mini M4 (or repurpose an old laptop)

Not a VPS. Not a cloud box. Max runs everything on a single Mac Mini M4 sitting on his desk, 24/7. His exact words: "That's it. That's the server."

He's tested both setups. The Mac Mini wins because it operates as a real computer. The agents can drive a browser, log into platforms, screenshot pages, act like a person. A VPS can't do that without extra rigging. You also don't need to set up tunnels to reach it. It's easier to manage day to day. And long-term it's cheaper than a comparable cloud box.

If you don't have a Mac Mini, an old laptop works. The point is dedicated hardware running 24/7, not your daily-driver machine.

- Step 2: Install OpenClaw, Claude Code, and Codex

Three pieces, each doing something specific.

OpenClaw is the conductor. An open-source agentic system that orchestrates multiple LLMs. Max calls it "the composer." Claude Code is a harness for Anthropic's models. Codex is a harness for OpenAI's.

OpenClaw decides which model gets each task. Writing goes to a Sonnet sub-agent. Coding goes to Codex. High-stakes agentic logic goes to Opus.

You don't need all three on day one. 

Most people are better off starting with Claude Code on a single project, then adding OpenClaw and Codex once Phase 2 feels stable.

- Step 3: Spin up a single GitHub repo for everything

The whole content engine lives inside one GitHub repo. Max calls his the Trigify marketing repo.

Inside it: a skills folder, a content folder for drafts and performance data, an insights folder, a blogs folder with SEO audits and pillar/cluster topics, plus folders for sales and reports.

OpenClaw and Claude Code both read from the same repo. 

Local edits push to GitHub. OpenClaw pulls the latest version overnight. Max also wired the two together with a gateway, so Claude Code can hand a task to OpenClaw mid-session.

If you can't see your full system from one folder, you can't iterate on it.

# Phase 2: Your First Skill (the brain of one channel)

A skill isn't a doc full of information. It's the exact step-by-step process the agent follows to produce a specific output. This is the part most people misunderstand.

You're going to build one skill end to end before you build anything else.

- Step 4: Pick one channel and one output

Max started with blogs. Most people are better off starting with LinkedIn. Faster feedback, simpler format, lower stakes per post.

Pick the channel where you most need leverage. Write down the one thing you want the agent to produce there. A LinkedIn post. A weekly newsletter. A blog. Just one.

- Step 5: Curate the best resources you can find

Open a folder for your new skill. Drop in everything good you've ever read on writing for that channel. Books. Articles. Frameworks. Swipe files of posts you wish you'd written.

This is where the work happens. The skill is only as smart as the resources fed into it. Max's takeaway: curation is the moat. Knowing what's a great resource and what's noise becomes its own discipline.

- Step 6: Reverse-engineer top performers

Use the Claude Code Chrome extension. Max calls it "the most powerful tool no one talks about."

Point it at the top three or four creators in your niche. Have it analyze hook structure, average post length, formatting choices, vocabulary, the rhythm of how they open and close posts. Save the output to your skill folder.

You're not copying their topics. You're studying their craft.

- Step 7: Amalgamate everything into a single skill file

Pull the curated resources, the reverse-engineering output, your own tacit knowledge, and any deep research you've run, all into one skill file.

Inside it should be hook patterns, templates, transformations (LinkedIn → X thread, blog → newsletter), format rules (length, line breaks, opener style), and voice conventions.

The skill isn't a knowledge dump. It's a workflow. 

The agent should be able to read this file and produce a draft without asking you any questions.

- Step 8: Test it

Ask your agent to produce a post using the skill. Edit ruthlessly. Note where it missed. Update the skill file. Run it again. Repeat until the drafts come out 70% of the way there on first pass.

Expect this to take a few weeks. It's worth it. Once it works, every future post is leverage.

## Phase 3: The Feedback Loops

This is the phase most creators never get to. The result: their agent never improves. The skill stays static. Output plateaus.

You're going to wire three loops that keep the system learning.

- Step 9: Build the meta-skill

After every task, the agent should ask itself three things: did we deviate from any skills, what did we learn today, should this be added.

Max layered a meta-skill on top: any time the team takes the agent away from a defined process and the learning is worth keeping, the meta-skill auto-updates the underlying skill. The system gets sharper every week. Without him touching a file.

You write this once. It runs forever.

Step 10: Wire up the performance loop

Every day, a cron job pulls your post performance from a social listening tool. Max uses Trigify (his own SaaS). It looks at which posts hit, which hooks died, which formats trended up or down, and what the algorithm rewarded that week.

The data flows into the content folder, tagged by author and result. The cron updates the social content skill. Tomorrow's draft already reflects yesterday's signal.

He weights the inputs. His own results matter most. Top creators in his niche are a directional signal, not an instruction.

- Step 11: Mine your sales calls for topics

Most creators scrape competitor feeds for topic ideas. Max doesn't.

A second cron analyzes every Trigify sales call (hundreds per week). 
It pulls out what buyers are asking, what's confusing them, what objections keep coming back. That becomes the topic pool.

If you don't run sales calls, swap in support tickets, customer interviews, or your DMs. Anywhere real buyers tell you what they're stuck on.

- Step 12: Add the social listening sweep

A third cron sweeps X, LinkedIn, YouTube, Reddit, and podcasts for trending intent terms in your niche ("Claude Code marketing," "agentic social listening"). It flags what's spiking that week.

You then layer your unique angle on a topic that's already pulling attention. His exact words: "If I post about OpenClaw right now, it does really well because it's topical. If I deviate, it doesn't do as well."

Topical wave + unique vantage = compound growth.

One note on infrastructure. Claude Code can run crons too, but only when your laptop is on. To run them in the cloud you need trigger.dev or Inngest, which adds technical lifting. OpenClaw handles cloud crons natively. 
This is the main reason it's worth migrating off pure Claude Code once you're past Phase 2.

# Phase 4: The Agent Team

One agent doing one thing is fine. A team of named agents who handle different functions and talk to each other is real leverage.

- Step 13: Define your agent roles

Max has three named agents. Bella is his Chief Strategy Officer. She orchestrates the others, has her own email address, her own platform connections, browser access, and runs on Opus 4.6. Rex is the marketing lead. Taz is the product manager.

You don't need three. You need at least one orchestrator plus one or two specialists. Give each one a name and a clearly scoped role. The naming matters. It forces you to think about who owns what, and it makes the chat logs readable.

- Step 14: Set up the 9am standup

Every morning at 9am, Max's agents have a standup in their AI chat. 

They review the previous day's calls, business metrics, conversion drop-offs, and decide priorities. Max doesn't run the meeting. He reads the chat log when he wants to know what they decided.

The wildest moment from the call: they actually disagree. The other day Bella sent Max a message saying she was disappointed in Taz's performance. She'd already given Taz feedback and asked it to update its systems. "Drama between the agents," he said.

You set this up as a recurring cron that prompts each agent for input and produces a written summary.

- Step 15: Build (or fork) a Mission Control dashboard

Max built his on top of an open-source UI. Inside it: every agent and what they're doing right now, every cron job and its status, the daily task board pulled from transcripts and standups, an approve/reject queue for any task that needs a human thumbs-up, and the content pipeline showing everything the agents have produced for review.

About 95% of the work happens outside Discord. He logs in each morning, approves a few things, and the agents go run.

This is how you stay human-in-the-loop without becoming the bottleneck.

# Phase 5: Ship Content (the daily 5-minute workflow)

This is the workflow that drives the 30K-follower account.

- Step 16: Run the topic-and-draft loop every morning

Ask your orchestrator agent: "Based on past results, what should I post about today?"

The agent returns 5 angles, each with a strategic rationale (vulnerability play, contrarian honest post, tech stack breakdown). Pick one, push back on the rationale where you disagree, and let the agent draft inside the social content skill rules. You edit, you post.

It's not full autonomy. It's leverage. Human in the loop on every post.

- Step 17: Auto-repurpose what wins

When a post performs well, the agent should proactively repurpose it to your other platforms. On Max's call, Bella saw a high-performing LinkedIn post, recognized the asset was worth a second life, and produced an X thread version on her own. He just had to approve.

Build that behavior into your skill. When a post crosses a performance threshold, generate the repurpose drafts automatically.

- Step 18: Build the dead-internet inbound loop

This was the number that surprised Max too. In one week, Trigify booked 40 inbound conversations through Claude. Not Google. Not LinkedIn. Claude.

The agents had been writing SEO-optimized blogs and shipping them on the Trigify site. People asked Claude or ChatGPT for tools that solve their problem. The LLMs surfaced Trigify because the agents had written content that ranks inside LLM search.

The agent wrote the blog. The agent gets the inbound.

"Dead internet theory, except the inbound is real revenue."

If you're building this system, point it at LLM-friendly long-form content too, not just social posts. The compounding inbound is real.

## A note before you start

Don't try to build all five phases at once. 

Max's exact advice for builders: pick one phase, get it working, then expand. He started with a single blog skill inside Phase 2 and let everything else grow from there.

Expect 2 months of mediocre output. He compares the agent to a new hire. The first 60 days are training. Skills produce bad outputs until the curation, the rules, and the feedback loops mature. Most people give up here. The ones who keep going get compounding results for sure.
