## The CLAUDE.md Template (4 Rules)

In late January 2026, [[andrej-karpathy]] posted a thread complaining about how Claude writes code. Three failure modes:

1. Silent wrong assumptions
2. Over-complication
3. Orthogonal damage to code it shouldn't have touched

Forrest Chang read the thread and packaged Karpathy's complaints into a 4-rule CLAUDE.md file. The repo hit:
- 5,828 stars in the first day
- 60,000 bookmarks in two weeks
- **120,000 stars** by May 2026

## The 4 Rules

1. **Think Before Coding** — State assumptions explicitly, surface tradeoffs, ask before guessing
2. **Simplicity First** — Minimum code that solves the problem, no speculative features
3. **Surgical Changes** — Touch only what you must, don't "improve" adjacent code
4. **Goal-Driven Execution** — Define success criteria, loop until verified
