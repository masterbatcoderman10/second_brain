# David Bayendor (@bayendor)

X creator who documented a production Hermes Agent stack built around three layers: Hermes Agent for execution, Honcho for persistent memory, and Hermes-LCM for measurement and control.

## Key Work

**Hermes + Honcho + LCM Stack** (April 2026): Built a production-ready Hermes Agent stack with three distinct layers — Hermes Agent (execution), Honcho self-hosted (persistent memory + user modeling), and Hermes-LCM (measurement/control). The goal was "result by data": repeatable, auditable agent outputs rather than vibe-based automation. The stack implements dual-peer memory modeling (user + AI peers), 28 permissions, 12 groups, 21 CSRF-protected endpoints, and a password-gated admin surface.
