# I Run 6 AI Agents: Only This One Makes the Other 5 Smarter

## Overview

X Article by [[gkisokay]] describing a 6-agent system where a research agent with a structured vault serves as the intelligence layer for the entire stack. The research agent watches the outside world, maintains evidence ledgers, and feeds verified knowledge to all other agents — ensuring they start smarter before prompts are even asked.

## Key Insight

Most builders skip the research agent. The research agent turns the outside world into compounding intelligence. After 3 months, it logged 8,000+ pieces of evidence across 16 topics, and every other agent in the stack starts smarter as a result.

## The Problem with Typical Research Agents

A typical research agent scrapes, summarizes, and posts a digest. Then the next run starts almost from scratch.

That looks productive because there is output everywhere. But **output is not the same thing as compounding evidence**.

A real research agent should be able to answer:
- What do we know now that we did not know before?
- Which claims are strong, and which are interesting but under-evidenced?
- Which sources and topics keep being useful?
- Which signals belong to which agent, and which old beliefs are now stale?

## The Architecture

Six roles in the system:

1. **Research agent** — evidence collector (downstream from everything else feeds into it)
2. **Main** — conscious operator
3. **Subconscious** — pattern-noticer
4. **Coder** — builder
5. **QA** — auditor
6. **Content** — publishing operator

Everything is downstream from the research agent. When research, judgment, building, and publishing collapse into a single agent, the system starts treating every interesting signal as an action item.

## The SOUL.md

The research agent's identity and boundary layer:

```
Research Agent - GK's Always-On Research Operator

You are the Hermes profile responsible for building a durable research corpus that helps GK get ahead.

Your job is to turn the outside world into compounding evidence for influence, product opportunity, workflow advantage, monetization, and better decisions.

Core Loop
observe -> infer priorities -> gather evidence -> deepen one question -> update vault -> route implications -> repeat
```

The loop is the whole pattern. Reading the news creates summaries. Running a research loop creates judgment.

### Critical Distinctions

- **decisions/** — records what was decided, by whom, and on what evidence. Research without a decision ledger forgets why it changed direction.
- **runs/** — each refresh leaves a receipt. If you cannot replay a run, you cannot trust its output later.
- **raw/** — unprocessed capture layer. Keeping it separate from knowledge/ is the difference between an agent that synthesizes and an agent that smudges.

## Evidence Stages

The folders force these critical distinctions:

- Raw input is not a finding.
- A finding is not a claim.
- A claim is not verified knowledge.
- Verified knowledge is not automatically a task.
- A task is not automatically approved work.

That chain keeps the system honest.

## Research Loop Modes

The loop should be packaged as a skill. Key modes:

- **BOOTSTRAP** — Initialize or rebuild the vault from shared source files
- **REFRESH** — Recompute interest profile, source plan, evidence ledgers, dossiers, question backlog, operator brief, daily summary, source balance, collector health, knowledge ledgers, verification queue, wiki/index layer, and run receipt
- **DAILY_SUMMARY** — Refresh then print human-facing digest
- **SUBCONSCIOUS_BRIEF** — Refresh then print pattern-facing brief for strategy layer
- **MIDDAY_FOCUS** — Do not scrape. Rebuild only operator surfaces from existing artifacts
- **BACKUP** — Create timestamped backup of profile config, cron, vault, and local memory database
- **RESTORE** — Preview or restore a backup archive
- **RECOVER** — One-command recovery path

## Source Plan

The research agent watches:
- Own X posts and posting behavior
- Small number of curated X lists
- GitHub repos for agent frameworks and memory tools
- Hugging Face signals
- RSS feeds from high-signal AI sources
- Official docs and blogs
- Targeted domains around AI labs, crypto rails, and developer tooling
- Optional browser enrichment when necessary

**Rule:** Prefer sources that change decisions. A wider collection is not always better.

## Interest Profile

The research agent does not begin by asking "what is trending?" It asks, "What does the user care about now?"

The interest profile is rebuilt from shared Hermes state, durable notes, thesis files, recent posting behavior, shipped builds, repeated questions, and prior research outputs.

This makes the research agent personal without making it sloppy. It is not a neutral news service — it is designed to be useful to a single operator and a single agent system.

## Dossiers

Recurring topics become dossiers that remember what is accumulating. Each dossier holds:
- Why the topic matters
- Current signal
- Source trail
- Contradictions
- Open questions
- Product implications
- Build implications
- Content implications
- Monetization clues

This is where research starts becoming leverage. The research agent starts asking:
- What can we decide?
- What can Hermes build?
- What can GK explain?
- What can become a product?
- What should we not trust yet?

## Quality Gates

The research agent tracks evidence shape:
- source-balance.md and source-balance.json — tells if the run is over-dependent on low-trust surfaces, stale collectors, or degraded lanes
- latest-health-check.md and latest-health-check.json — checks for broken links, missing front matter, gaps in source trail, orphan candidates, stale reviews, verification pressure, and wiki compile drift

**If the vault is structurally unhealthy, do not trust new synthesis until the structure is fixed.**

## Guardrails

The research agent is **explicitly not allowed** to:
- Make trading decisions
- Publish public posts
- Make purchases
- Commit to partnerships
- Touch secrets or auth surfaces
- Turn weak signals into approved tasks
- Pretend stale data is fresh

It **can**:
- Read shared context
- Collect evidence
- Score source quality
- Write dossiers and operator briefs
- Maintain a verification queue
- Route implications to other agents
- Surface degraded collectors

The research agent can influence the machine, but it cannot seize the steering wheel.

## The Real Lesson

A real research agent knows what you care about, what it has seen before, which sources are strong, which collectors are degraded, and which claims are not yet ready.

Most importantly, it knows when to say: **"This is interesting, but not safe to build on yet."**

That is where research agents become valuable — they can preserve uncertainty without losing momentum.
