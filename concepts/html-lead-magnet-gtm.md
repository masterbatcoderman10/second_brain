# HTML Lead Magnet GTM

Using single-file HTML pages as go-to-market assets — replacing PDFs, Google Docs, and Notion pages with interactive, browser-native deliverables that signal customisation and effort.

## Core Thesis

**PDF says:** "We put in effort before we knew you existed. Nobody reads past page three."

**HTML says:** "We built this specifically for you. Open it in any browser. No login. No software."

The pre-demonstration model — creating custom value before a sales call — is more effective when the deliverable is an HTML file rather than a PDF. HTML can carry 8 types of information PDF cannot: interactive pricing, tabbed navigation, embedded data visualisations, prospect-specific language, and more.

## Six GTM Use Cases

1. **Lead Magnets** — Interactive wikis scoped to prospect's industry; built with Claude Code in one session; prospects bookmark rather than delete
2. **Custom Proposals** — Tabbed, navigable documents scoped to prospect's business; interactive pricing toggles; visual week-by-week workflows; prospect-specific FAQ
3. **Pre-Demonstration Deliverables** — Custom audits, competitive analyses, market research reports, scoring rubrics — built from prospect's own website, LinkedIn, and industry data in minutes
4. **Interactive Guides** — Self-contained prospect-specific reference materials that get bookmarked
5. **Competitive Analysis Documents** — Live-looking reports built from public data
6. **Nurturing Sequences** — HTML sequences that progressively demonstrate capability

## Why HTML Beats PDF

| Capability | PDF | HTML |
|---|---|---|
| Opens in any browser | No | Yes |
| No login required | No | Yes |
| Interactive elements | No | Yes |
| Updates after sending | No | Yes |
| Feels custom-built | Rarely | Always |
| Production cost | Hours | Minutes |

## Implementation

Claude Code can produce a publishable HTML lead magnet in one session. The deliverable is a single `.html` file — one URL, one attachment, distributed via cold email.
