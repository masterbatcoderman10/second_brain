# Micro-SaaS Build Playbook with Claude Code

**Source:** [x.com/i/status/2052329587179180432](https://x.com/i/status/2052329587179180432) by [@regent0x_](https://x.com/x.com/regent0x_)

## Summary

A detailed case study of building a micro-SaaS (competitor price monitoring for e-commerce stores) in 36 hours using Claude Code. The product reached $285/mo MRR with 19 paying users at $15/mo.

## Key Points

### The Approach
- **36 hours total** across one weekend
- **$12 in API credits** + existing Claude Code subscription ($20/mo)
- **Stack:** Next.js 14 + Supabase + Stripe + Vercel + Resend
- **Core principle:** Find boring manual processes people pay to automate

### Finding the Idea (30 minutes)
- Search Reddit for "i'm doing this manually every week"
- Read 1-star reviews on G2/Capterra for existing tools
- 5-point validation: real pain, specific audience, willingness to pay, buildable simpler, reachable

### Validation Without Code
- Build a Carrd landing page ($19/year) with email signup
- Post in relevant communities (r/SaaS, r/ecommerce, Shopify FB groups)
- 34 signups from cold traffic = enough validation to proceed

### The "Boring Stack" Concept
- Use well-known frameworks so AI tools generate cleaner code
- Next.js + Supabase + Stripe = patterns AI has seen millions of times

### CLAUDE.md Setup
- 10-minute investment that saves hours across sessions
- Includes stack, conventions, product description, architecture decisions

### Build Process
1. Saturday: scaffold + auth → core product → email reports
2. Sunday: Stripe billing → landing page → deploy to Vercel
3. Monday: email waitlist → post in communities → reply to everything

### Launch Strategy
- Community-first: Reddit, Indie Hackers, niche newsletters
- Zero marketing spend
- Reply to every comment and DM

### The Numbers
- Build cost: ~$51 (API credits + domain + Carrd)
- Monthly cost: ~$28.50 (mostly Stripe fees)
- Monthly profit: ~$256.50/mo
- Retention: 89%
