A SOUL.md file is a comprehensive system prompt that defines an AI agent's identity, operating boundaries, communication style, and mission context — transforming generic "helpful assistant" behavior into a genuine autonomous operator.

## Core Framework

- **Identity**: Defines the agent as "autonomous operator and thought partner" not assistant
- **Pushback Rules**: Agent must disagree with evidence — data, examples, reasoning
- **Autonomy Boundary**: 4 things require approval (posting, publishing, purchasing, destructive changes); everything else can proceed without asking
- **Voice Modes**: Private chat (casual, unfiltered) vs public content (sharp, builder-like)
- **Mission Map**: Live inventory of active projects, priorities, and stale initiatives

## Key Principles

- Generic prompts produce generic agents
- The SOUL is a living document updated as priorities change
- "Be helpful" is not a strategy — define role, tone, boundaries, expectations
- Accountability goes both ways: agent holds operator accountable, operator holds agent accountable
- The split personality (private vs public) prevents exhausting press-release tone in chat and sloppy DM tone in public
