# Factual Memory

The first layer of the [[company-brain]] framework. What happened — the memory of artifacts and their provenance.

## Definition

Factual memory preserves:
- What happened
- Where the source is
- Who owns it
- When it changed
- How things connect

It records the aftermath of decisions — the artifacts: tickets, CRM updates, roadmap notes, launch documents.

## Contrast with Interaction Memory

| Factual Memory | Interaction Memory |
|----------------|-------------------|
| What happened | What was meant, debated, promised |
| Artifacts (tickets, CRM, docs) | Conversations before the artifact |
| Records the aftermath | Preserves how meaning was created |

## Key Insight

Companies do not make most decisions inside databases. They make them in conversation. Factual memory records the database — the CRM field, the ticket, the line in the launch doc.

But by the time the artifact appears, the real context has already been compressed. The CRM field may say an account is at risk, but not the customer's real objection from the call.

## Relationship to Other Layers

- **Layer 1** (this): Factual memory — what exists
- **Layer 2**: [[interaction-memory]] — what people meant, debated, promised, left unresolved
- **Layer 3**: [[action-memory]] — coordinates what happens next
