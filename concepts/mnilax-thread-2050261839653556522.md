# Mnilax 430-hour Claude Code audit

> 90-day self-audit by [[mnilax]] (Mnimiy). 430 hours, 6M input tokens, $1,340 API spend. 73% of tokens were overhead (not productive work). 9 invisible patterns identified with 30-second fixes each.

## Summary

[[mnilax]] tracked every Claude Code session for 90 days using an HTTP proxy between Claude Code and the Anthropic API. Categorized tokens into: productive, cache hit (free), cache miss (paid), conversation history re-read, hook injection, skill loading, tool use overhead, extended thinking, and CLAUDE.md baseline.

**Key finding:** Only 27% of tokens were productive. After fixing the 9 overhead patterns, productive share rose to ~65%.

### The 9 overhead patterns (ranked by cost)

| # | Pattern | % of total tokens | Fix |
|---|---------|-------------------|-----|
| 1 | CLAUDE.md bloat | ~14% | Cut combined CLAUDE.md to <1,500 tokens; move rules to project-level or skills |
| 2 | Conversation history re-reads | ~13% | Hard cap at 20 messages; use /compact not /clear; edit prior instead of follow-up |
| 3 | Hook injection waste | ~11% | Audit UserPromptSubmit hooks; disable any not justifyable per-task |
| 4 | Cache misses on resume | ~10% | Upgrade to 1-hour cache lifetime; use ping hotkey to keep cache warm |
| 5 | Skill loading on irrelevant tasks | ~7% | Audit which skills are actually invoked; disable rest |
| 6 | "Just in case" tool definitions | ~6% | Disable unused MCPs; keep only 3 always-on |
| 7 | Extended thinking on simple questions | ~5% | Default OFF; toggle ON per-message (Alt+T) only when needed |
| 8 | Wrong-direction generation | ~4% | Use Cmd+. / Ctrl+. to stop immediately when drifting |
| 9 | Plugin auto-update redundancy | ~3% | Kill SessionStart "loaded successfully" notifications |

### What didn't work

- Switching to Haiku for simple tasks (~3% reduction only)
- Aggressive /clear between every task
- Disabling all skills entirely (wrote same instructions manually)
- Off-peak hours scheduling
- Subscription downgrade

### Methodology

- HTTP proxy between Claude Code and Anthropic API
- Logged every request: full payload, response, token counts (input/output/cache), latency, model
- 90 days, 430 hours of active work

### Accompanying audit script

A bash script that flags all 9 patterns at once for users to run on their own setup.

### Related findings

- Anthropic acknowledged usage limit problem in late March 2026
- Peak-hours quota change explained part of it
- Two prompt-cache bugs (cache layer) inflated costs 10-20x for some sessions (patched)
- Downgrading to v2.1.34 helped some users
