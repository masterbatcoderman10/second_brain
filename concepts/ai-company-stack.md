# AI Company Stack

The combination of [[paperclip-orchestrator]] + [[hermes-agent]] as a "one person into a full AI company" system. Thesis by [[juliangoldie]]: orchestration handles task routing and multi-agent coordination, while Hermes adds memory and skill building for continuity across sessions.

## How It Works

**Paperclip** handles:
- Task orchestration and routing
- Multi-agent coordination
- Workflow structure and continuity

**Hermes Agent** handles:
- Persistent memory across sessions
- Skill building from repeated tasks
- Reducing the "start from zero every session" problem

## Why It Matters

Most automation stacks fail when parts don't communicate. The Paperclip + Hermes combination addresses:
1. **Scattered outputs** → Hermes memory creates continuity
2. **Duplicated effort** → shared context reduces repetition
3. **Constant retraining** → skill building compounds over time

## Use Cases

- **Content production**: research → draft → review → publish pipeline
- **Outreach**: context-aware messaging that improves over time
- **Operations**: reducing daily operational noise for solo operators
